# Architecture

## What this app is

Mobile-first PWA for dispatch operations. Workers log incidents with voice notes, photos, video, and files. **Zero backend** — all data is on-device in IndexedDB. Installable (PWA), works offline after first load.

## Data flow

```
User action (tap/record/type)
        │
        ▼
React component (useState local draft)
        │  onBlur / explicit save
        ▼
lib/db.ts  ─────►  IndexedDB (idb)
        │
        ▼
TanStack Query cache invalidation
  qc.invalidateQueries(...)
        │
        ▼
Re-render with fresh data
```

All blob data (images, audio, video) lives directly in the `attachments` array on the `Entry` object, stored as `Blob` in IDB. No separate blob store.

## Build & deployment

### GitHub Pages (primary)

```bash
GITHUB_PAGES=true npm run build
# output: dist/client/
# 404.html is copied from index.html for SPA fallback routing
```

CI: `.github/workflows/deploy.yml` — triggers on push to `main`.

### Cloudflare Workers (alternative)

```bash
wrangler deploy
# entry: src/server.ts
# serves TanStack Start SSR
```

### Local dev

```bash
npm run dev
# Vite dev server, middlewareMode: true
# Hot reload via @vitejs/plugin-react
```

## Routing

File-based routing via `@tanstack/router-plugin/vite`. The plugin auto-generates `src/routeTree.gen.ts` — **never edit that file manually**.

`createRootRouteWithContext<{ queryClient: QueryClient }>()` — `QueryClient` is injected at router creation time and available in all routes via `Route.useRouteContext()`.

`scrollRestoration: true` is enabled on the router.
`defaultPreloadStaleTime: 0` — routes always re-fetch on navigate.

## Key design decisions

1. **No backend** — deliberate. All data local. No auth, no sync, no server costs.
2. **Blobs in IDB** — simple, self-contained. IDB handles large binary data well. No URL expiry issues.
3. **Prerender enabled** — all routes are statically generated at build time, making GitHub Pages deployment trivial and load times instant.
4. **Image downscale on capture** — `lib/image.ts` downscales to 1800px max edge at 0.85 JPEG quality before storing, preventing memory crashes on mobile.
5. **Reminders are setTimeout-only** — no Push API, no background sync. Notifications only fire while the tab is open. Accepted limitation for simplicity.
6. **Single dark theme** — no light mode, no theme switching. Midnight Indigo only.
